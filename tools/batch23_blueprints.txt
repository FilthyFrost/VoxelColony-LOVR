**IMPORTANT NOTES:**

1. **Stable 2** has limited per-block detail because the WebFetch tool could not fully extract its 16x16 grid data (it uses white terracotta instead of cobblestone, which is the key differentiator from Stable 1). The material summary is provided instead.

2. **Temple 1** is a large church-like structure (5x9 footprint, 12 layers tall) with a tower section. The upper layers (y=4 through y=11) are mostly hollow cobblestone walls with ladders going up. The WebFetch tool returned all cobblestone for the foundation layer which appears to be over-flattened -- in reality, the upper layers have hollow interiors and the tower narrows.

3. The wiki row/column mapping: wiki rows map to Z coordinates (top row = z=0), wiki columns map to X coordinates (left column = x=0). Each "Layer N" on the wiki maps to y = N-1 in these blueprints.

4. **Stair facing directions** are inferred from their position in the structure (edge stairs face outward for rooflines). The wiki uses sprite images for facing which the text-based extractor could not always resolve.

5. Some structures have sub-ground layers (Layer 0 on the wiki = y=-1 in these blueprints) for terrain features like farmland, water, and grass blocks.
