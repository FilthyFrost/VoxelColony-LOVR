# Voxel Colony - Game Design Document

## 游戏概述

| 字段 | 值 |
|------|-----|
| 游戏名称 | Voxel Colony（体素殖民地） |
| 类型 | 街机式殖民地服务 + 点击投放 + 赌博 |
| 平台 | Desktop (LOVR 0.18) |
| 目标用户 | 中国玩家，喜欢快节奏、轻度策略、解压/搞笑向 |
| 单局时长 | 10-15 分钟 |
| 视角 | 上帝视角（自由飞行摄像机） |
| 语言 | 中文（NPC对话、UI、Game Over评价） |

## 核心幻想 (Core Fantasy)

**反转主仆关系** — 传统游戏里玩家是上帝、NPC是工具人。本游戏反转：

- NPC 是暴君、甲方、大爷
- 玩家 是苦力、乙方、搬砖工

NPC 对玩家骂骂咧咧、呼来喝去，玩家必须疯狂点击鼠标投放物资来满足它们不断升级的需求。满足得慢，NPC就会死亡。所有NPC死亡 = 游戏结束。

## 三个核心爽点

### 1. 被暴君统治的喜剧感
NPC用中文骂骂咧咧地提需求，语气欠揍但不恶毒。玩家的乐趣来自"我居然在被一个像素小人使唤"的荒谬感。类似 Untitled Goose Game 的幽默气质。

### 2. 疯狂点击的紧迫感
多个NPC同时尖叫，玩家手忙脚乱地切换材料、点击投放。节奏像 Overcooked — 知道该做什么，但做不过来。升级系统让后期每一次点击都很爽（一点下去哗啦啦一片方块雨）。

### 3. 赌博的刺激感
被迫和恶魔赌博。赢了获得强力buff（方块磁铁、连射模式），输了灾害降临（暴风、骤降气温）。世界不暂停 — 你在打牌的时候NPC在背景里饿着肚子尖叫。

## 与现有代码的关系

当前代码已实现的技术基础：
- 体素世界渲染（GPU实例化）
- NPC寻路（A* 3D）
- 建筑模板系统（40+模板）
- NPC建造行为（协作建造、步骤管理）
- 生存系统（饥饿、温度、体力、社交）
- 昼夜循环

需要新增/修改的系统：
- 需求驱动系统（替代当前的Utility AI自主决策）
- 中文对话系统
- 玩家投放升级系统
- 恶魔赌博系统
- 灾害系统
- Game Over / 计分系统
- HUD重构
