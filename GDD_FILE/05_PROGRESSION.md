# 05 - 游戏进程与难度曲线 (Progression)

## 设计意图

游戏从"轻松的教学"平滑过渡到"手忙脚乱的混乱"。玩家不应感到突然的难度跳跃，而是逐渐被"温水煮青蛙"式地推入越来越大的压力中。

## 阶段详细设计

### Phase 1: 教学期（0:00 - 2:00）

**NPC数量**: 1
**活跃需求**: 1（同一时间只有1个需求）
**投放等级**: Lv.1（每点1个）

| 时间 | 事件 | 玩家体验 |
|------|------|----------|
| 0:00 | 1个NPC出现在地图中央，喊出第一个建造需求 | 学习操作 |
| 0:00-0:30 | NPC的气泡带操作提示 "[←→选材料] [点击投放]" | 引导投放 |
| 0:30-1:00 | 玩家逐个投放材料，进度条更新 | 理解反馈循环 |
| ~1:00 | 材料凑齐，NPC开始建造 | 休息+观赏 |
| ~1:20 | 建造完成，NPC喊"要伴侣" | 学习N键生成NPC |
| ~1:30 | 玩家按N，新NPC出现 | 成就感 |
| ~1:40 | 两个NPC同时喊食物需求 | 学习切换到苹果+投放食物 |
| ~2:00 | 食物投放完毕 | 完成教学，**投放升级到Lv.2（×3）** |

**节奏**: 松弛，给玩家充足时间学习。NPC的催促台词切换较慢（8秒一句）。

### Phase 2: 扩张期（2:00 - 5:00）

**NPC数量**: 2-3
**活跃需求**: 1-2（开始重叠）
**投放等级**: Lv.2（每点3个）

| 时间 | 事件 | 难度变化 |
|------|------|----------|
| ~2:00 | NPC喊建造仓库需求 | 第一个"大工程" |
| ~2:30 | 第2个NPC也开始喊食物需求 | 首次面临多需求 |
| ~3:00 | **第一个恶魔出现** | 首次赌博体验 |
| ~3:30 | 赌博结束（无论胜负） | 要么获得buff，要么面对灾害 |
| ~4:00 | 再来一个伴侣需求 | NPC数到3 |
| ~4:30 | 新NPC喊建房需求 | 多建筑同时进行 |
| ~5:00 | **投放升级到Lv.3（×6）** | 应对提升的需求量 |

**节奏**: 中等，开始感到"有点忙"。NPC催促加快（6秒一句）。第一次夜晚可能让无庇护NPC进入CRITICAL。

### Phase 3: 压力期（5:00 - 10:00）

**NPC数量**: 4-6
**活跃需求**: 2-4（频繁重叠）
**投放等级**: Lv.3-4（每点6-12个）

| 时间 | 事件 | 难度变化 |
|------|------|----------|
| ~5:00-6:00 | 多个NPC同时有不同需求 | 必须做优先级选择 |
| ~6:00 | **第二个恶魔出现** | 赌博频率加快 |
| ~7:00 | 可能经历第一次NPC死亡 | 压力骤增 |
| ~8:00 | NPC要求更大的建筑（medium_house, library等） | 材料需求量大增 |
| ~9:00 | **第三个恶魔出现** | 灾害可能叠加 |
| ~10:00 | **投放升级到Lv.4（×12）** | 勉强跟上需求 |

**节奏**: 紧张，频繁切换材料类型。NPC催促很快（4秒一句），多个NPC的气泡此起彼伏。

### Phase 4: 混乱期（10:00 - 15:00+）

**NPC数量**: 7-10
**活跃需求**: 3-5（几乎持续）
**投放等级**: Lv.4-5（每点12-24个）

| 时间 | 事件 | 难度变化 |
|------|------|----------|
| ~10:00-12:00 | 需求不间断涌来 | 几乎没有休息窗口 |
| ~11:00 | **恶魔每2分钟出现** | 赌博成为常态 |
| ~12:00 | 灾害可能同时存在多个 | NPC死亡频率增加 |
| ~13:00 | 开始出现NPC连续死亡 | 游戏进入倒计时 |
| ~15:00+ | 最后一个NPC死亡 | Game Over |

**节奏**: 极度混乱，是否还能完成翻看的需求。升级到Lv.5(×24)也只是勉强维持。这个阶段的设计目标是"玩家知道要输了，但想多撑一秒"。

## 难度调控参数

### NPC生存时间基准

| 威胁 | 无干预存活时间 | 说明 |
|------|---------------|------|
| 饥饿 | ~90秒 | 从满到0的衰减时间 |
| 夜晚冻死 | ~45秒 | 无庇护NPC在夜间的存活时间 |
| 体力耗尽 | 不致死 | 昏倒后间接加速饥饿/冻死 |

### 需求频率

| 阶段 | 新需求产生间隔 | 同时活跃需求上限 |
|------|---------------|-----------------|
| Phase 1 | 完成后才产生下一个 | 1 |
| Phase 2 | 10-15秒 | 2 |
| Phase 3 | 6-10秒 | 4 |
| Phase 4 | 3-5秒 | 5 |

### 建筑模板解锁

| 阶段 | 可用模板池 | 平均方块数 |
|------|-----------|-----------|
| Phase 1 | small_house_1~8, small_cozy_house, start_house | 50-80 |
| Phase 2 | + cozy_cabin, lamp_1, small_farm_1, meeting_point_1~2 | 80-120 |
| Phase 3 | + medium_house_1~2, butcher_shop, fisher_cottage, tannery | 120-200 |
| Phase 4 | + big_house_1, library_1~2, temple_1~2, stable_1 | 200-400 |

## NPC数量增长

NPC不会自动增长。只有当NPC发出"伴侣需求"且玩家按N生成时才增加。

伴侣需求的产生条件：
- 已完成的房子数量 > 当前NPC数量 / 2
- 当前NPC < 10（硬上限）
- 没有活跃的伴侣需求

这意味着玩家必须先建足够的房子，NPC才会要求更多人。建筑 → 人口 → 更多建筑需求 → 更多人口的正反馈循环。

## 难度的自然调节

设计中内置了两种自然的难度调节：

### 正反馈（越强越强）
- 更多NPC → 更多NPC帮忙建造 → 建造更快
- 升级投放 → 每次点击效率更高 → 能更快满足需求

### 负反馈（越强越难）
- 更多NPC → 更多食物需求 → 更多点击
- 更多建筑 → 更大的灾害损失 → 更多修复工作
- 恶魔频率加快 → 灾害可能叠加

两者平衡使得游戏不会太早结束也不会无限持续，最终负反馈占上风导致Game Over。
