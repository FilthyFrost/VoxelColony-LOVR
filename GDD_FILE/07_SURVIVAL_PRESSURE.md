# 07 - 生存压力系统 (Survival Pressure)

## 设计意图

NPC的死亡必须是玩家**直接能感知和干预**的。每次NPC死亡，玩家都应该清楚地知道"是因为我不够快"。生存压力是驱动点击紧迫感的引擎。

## MVP模式移除

当前代码 `npc.lua:166-170` 锁定了所有生命值：
```lua
-- MVP MODE: lock vitals so NPCs never die during testing
self.hp = self.cfg.HP_MAX
self.hunger = self.cfg.HUNGER_MAX
self.stamina = self.cfg.STAMINA_MAX
self.temperature = self.cfg.TEMP_MAX
```

**删除这4行**，让生存系统生效。

## 三条命脉

### 1. 饥饿（主要死因）

**设计原则**：饥饿是最直接的压力源。玩家扔苹果 → NPC吃 → 续命。不扔 → 饿死。

```lua
-- 关键参数
HUNGER_MAX = 100
HUNGER_DECAY = 0.2          -- 每秒衰减0.2
-- → 从满到0需要 100/0.2 = 500秒 ≈ 8.3分钟（太慢）

-- 新参数（加速衰减，让游戏节奏更紧）
HUNGER_DECAY = 1.1          -- 每秒衰减1.1
-- → 从满到0需要 100/1.1 ≈ 90秒 = 1.5分钟

HUNGER_EAT_RESTORE = 40     -- 每个苹果恢复40（保持不变）
-- → 一个苹果续命约36秒
```

**饥饿状态分级**：

| 饥饿度 | 状态 | 表现 |
|--------|------|------|
| 100-50 | 正常 | 无特殊表现 |
| 50-30 | 饥饿 | 产生食物需求，气泡黄色 |
| 30-15 | 很饿 | 气泡变大+橙色，NPC移速-30% |
| 15-0 | 濒死 | 气泡红色脉冲，NPC移速-60%，屏幕边缘泛红 |
| 0 | 死亡 | NPC倒地，遗言台词，从世界移除 |

### 2. 温度（夜间威胁）

**设计原则**：温度只在夜间构成威胁，迫使玩家在天黑前建好庇护所。

```lua
-- 关键参数
TEMP_MAX = 100
TEMP_DECAY = 2.2            -- 夜间无庇护每秒衰减2.2（原1.0太慢）
-- → 从满到0需要 100/2.2 ≈ 45秒
TEMP_REGEN = 1.5            -- 白天恢复（保持不变）
TEMP_SHELTER = 4.0          -- 有庇护时恢复（保持不变）

-- 昼夜周期
DAY_LEN = 300.0             -- 5分钟一个完整周期
NIGHT_START = 0.75           -- 75%时进入夜晚
NIGHT_END = 0.95             -- 95%时天亮
-- 夜晚持续 = 300 × (0.95 - 0.75) = 60秒
```

**温度状态分级**：

| 温度 | 状态 | 表现 |
|------|------|------|
| 100-50 | 温暖 | 无特殊 |
| 50-30 | 发冷 | NPC身体颜色偏蓝 |
| 30-15 | 冻伤 | NPC速度-50%，气泡"冷死了！！" |
| 15-0 | 濒死 | 身体深蓝色，移速极慢 |
| 0 | 冻死 | 死亡 |

### 3. 体力/睡眠（间接威胁）

**设计原则**：体力耗尽不直接杀死NPC，但昏倒的NPC不吃东西不避寒，间接加速死亡。

```lua
-- 保持现有参数，但调整衰减速度
STAMINA_MAX = 100
STAMINA_DECAY_WORK = 0.3     -- 建造时每秒衰减
STAMINA_EXHAUSTED = 10       -- 低于此值强制昏倒

-- 睡眠恢复
STAMINA_REGEN_SLEEP_BED = 1.0     -- 有床：100秒恢复满
STAMINA_REGEN_SLEEP_INDOOR = 0.5  -- 有屋无床：200秒
STAMINA_REGEN_SLEEP_GROUND = 0.3  -- 露天：333秒
```

## 三命脉互锁（防退化策略）

玩家不能只靠一种资源通关：

| 如果只投食物不建房 | 白天活，夜晚冻死 |
|---|---|
| 如果只建房不投食物 | 有庇护但饿死 |
| 如果不投NPC | 单个NPC最终因需求加速而跟不上 |
| 如果不投建材扩建 | 新NPC没有庇护，夜间冻死 |

## 死亡流程

```lua
function NPC:die()
    -- 1. 播放死亡遗言
    self:setDialogue("DEATH")
    
    -- 2. 倒地动画（NPC躺下，持续2秒）
    self.dying = true
    self.dyingTimer = 2.0
    
    -- 3. 通知其他NPC（悲伤反应）
    for _, other in ipairs(self.allNpcs) do
        if other ~= self and not other.dead then
            other:setDialogue("COMPANION_DIED")
            other.moodValue = other.moodValue - 30
        end
    end
    
    -- 4. 标记死亡
    self.dead = true
    
    -- 5. 释放其建筑所有权（房子变成可用）
    if self.blueprint then
        self.blueprint.owner = nil
    end
    
    -- 6. 检查是否所有NPC都死了 → Game Over
    checkGameOver()
end
```

### 死亡视觉表现

```
死亡瞬间：
1. NPC身体变灰色
2. 头顶弹出遗言气泡（红色背景）："都...怪你..."
3. NPC慢慢躺下（复用现有睡眠动画）
4. 2秒后消失（淡出）
5. 地面留下一个小十字标记（5秒后消失）
```

### 屏幕死亡提示

NPC死亡时，屏幕顶部弹出红色横幅：
```
┌──────────────────────────────────────┐
│  ☠ 小蚁-3 饿死了... "都...怪你..."    │
└──────────────────────────────────────┘
```
持续3秒后消失。

## NPC初始状态

新生成的NPC不应该立即面临生存压力（否则刚生成就要死了）：

```lua
-- 新NPC的初始值
hunger = HUNGER_MAX * 0.8    -- 80%饱（有约72秒的缓冲）
temperature = TEMP_MAX       -- 满温度
stamina = STAMINA_MAX * 0.9  -- 90%体力
```

## 灾害对生存系统的影响

灾害（见 06_DEMON_GAMBLING.md）通过修改 Config 参数来加速生存衰减：

| 灾害 | 影响 | 持续 |
|------|------|------|
| 寒潮 | TEMP_DECAY × 3 | 30秒 |
| 饥荒 | HUNGER_DECAY × 2 | 45秒 |
| 暴风 | 摧毁方块（移除庇护） | 瞬间 |
| 末日风暴 | 以上全部 | 45秒 |

灾害结束后参数恢复原值。

## 调参指南

核心公式：**NPC无干预存活时间 = MAX / DECAY**

目标存活时间：
- 饥饿：~90秒（迫使玩家每90秒至少投1个苹果/NPC）
- 夜间冻死：~45秒（夜晚60秒，意味着无庇护NPC只能撑3/4个夜晚）
- 灾害中：存活时间缩短到原来的1/2~1/3

10个NPC时的食物需求：
- 每个NPC每90秒需要约2个苹果 → 10个NPC每分钟需要 ~13 个苹果
- Lv.5投放（×24）每次点击24个苹果 → 大约每分钟点击1次食物即可
- 这意味着后期食物不是主要瓶颈，**建材才是**（因为灾害破坏建筑需要修复）
