# 01 - 核心循环 (Core Loop)

## 基本循环

```
NPC产生需求 → 对玩家喊出需求 → 玩家点击投放材料 → 需求计数器更新
    ↑                                                      ↓
    |                                              需求满足？
    |                                             /         \
    |                                           否           是
    |                                           |            ↓
    |                                     继续喊(更急)   NPC开始行动
    |                                                    (建造/吃东西/迎接伙伴)
    |                                                        ↓
    |                                                   行动完成
    |                                                        ↓
    ←————————————————————————————————————————————  产生下一个需求
```

## 需求类型

| 需求类型 | 触发条件 | 玩家操作 | NPC行为 |
|----------|----------|----------|---------|
| **建造需求** | NPC无房/需要新建筑 | 投放指定数量的指定材料 | 拾取材料，按模板建造 |
| **食物需求** | 饥饿度 < 50% | 投放苹果 | 走过去吃掉 |
| **伴侣需求** | NPC独居，建完第一个房子后 | 按N键生成新NPC | 新NPC加入殖民地 |
| **扩建需求** | NPC数量增加，需要更多建筑 | 投放建材 | 建造新建筑（仓库、工坊等） |

## 一局游戏的完整流程

### Phase 0: 开场（0-10秒）
- 地图中央出现1个NPC
- NPC头顶弹出大气泡："喂！你！给我搬10个圆石来！快点！"
- 屏幕底部显示材料选择器 + 操作提示
- 玩家学习：按←→选材料，点击投放

### Phase 1: 教学期（0-2分钟）
- **需求1**：第一个房子的材料（由NPC自动选择模板，NPC根据性格选）
  - NPC持续喊材料需求，气泡显示进度 "圆石 3/10"
  - 玩家逐个点击投放
  - 材料够了 → NPC说"总算好了" → 开始建造
  - 建造过程中NPC头顶显示锤子图标，玩家可以观赏
- **需求2**：建完后NPC喊 "太无聊了！给我找个伙伴来！"
  - 屏幕提示 "[N] 生成NPC"
  - 玩家按N → 新NPC出现
- **需求3**：两个NPC同时喊 "饿了！给吃的！"
  - 玩家切换到苹果，投放食物

### Phase 2: 扩张期（2-5分钟）
- 2-3个NPC，需求开始重叠
- NPC们要求建仓库、建更大的房子
- 每满足一个完整建筑需求 → 玩家投放等级+1（1→3）
- **第一个恶魔出现**（约3分钟时）

### Phase 3: 压力期（5-10分钟）
- 4-6个NPC，需求密集
- 夜晚开始产生真正的生存压力（无庇护NPC会冻死）
- 恶魔定期出现（每2-3分钟）
- 投放等级继续提升（6→12）

### Phase 4: 混乱期（10-15分钟）
- 7-10个NPC，需求几乎无间断
- 多重灾害可能同时存在
- 投放等级高（12→24），但需求量也大
- 最终所有NPC死亡 → Game Over

## 核心节奏设计

**HIIT（高强度间歇训练）节奏**：

```
时间轴:
[疯狂点击20s] → [观看NPC建造15s] → [疯狂点击25s] → [赌博30s] → [灾害+疯狂点击30s] → ...
 ████████████    ░░░░░░░░░░░░░    ████████████████    ▓▓▓▓▓▓▓▓▓▓▓▓    ██████████████████
   爆发期            休息期            爆发期           紧张期          超级爆发期
```

- **爆发期**：NPC在喊，玩家疯狂点击投放材料
- **休息期**：NPC在建造，玩家观看（自然的手部休息）
- **紧张期**：恶魔赌博，玩家做决策
- **超级爆发期**：灾害后的补救，比普通爆发更紧急

关键设计原则：NPC的建造时间 ≈ 玩家投放时间 + 10秒。这确保玩家每次投放完毕后有约10秒的观赏/休息窗口。

## 游戏结束条件

- **失败**（唯一结局）：所有NPC死亡
- 没有"胜利"条件 — 这是一个生存型街机游戏，目标是活得尽可能久
- 死亡后进入 Game Over 评分画面（见 09_GAME_OVER_AND_REPLAY.md）

## 与现有系统的关键变更

| 现有行为 | 新行为 |
|----------|--------|
| 玩家选模板，一键丢所有材料 | NPC选模板，玩家逐个点击丢材料 |
| NPC用Utility AI自主决策做什么 | NPC根据需求系统被动响应 + 自主建造 |
| 材料无限（从天上凭空丢） | 材料仍然凭空丢，但需要玩家手动点击 |
| 没有游戏结束条件 | 所有NPC死亡 = Game Over |
| 没有升级系统 | 满足需求 → 投放等级提升 |
| 没有赌博/灾害 | 恶魔赌博 + 天气灾害 |
