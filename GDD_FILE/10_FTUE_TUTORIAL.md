# 10 - 新手引导 (First-Time User Experience)

## 设计意图

零教学UI、零弹窗教程。NPC的骂人台词本身就是教学。玩家在"被使唤"的过程中自然学会所有操作。

## 核心原则

1. **NPC嘴里说出操作提示** — 不是独立的教学弹窗
2. **一次只教一个操作** — 不要同时告诉玩家5个键
3. **做对了立即正反馈** — NPC的语气从骂变成嫌弃式肯定
4. **做错了有提示但不惩罚** — 前30秒NPC不会饿死（有保护期）

## 教学流程

### Step 1: 学习投放（0:00 - 0:30）

**触发**：游戏开始，1个NPC出现

**NPC台词**：
```
"喂！你！给我搬10个圆石来！快点！"
```

**屏幕底部同时显示操作提示**（嵌入在材料选择器上方）：
```
[←→ 选材料]  [鼠标左键 投放]
```

**细节**：
- 提示文字半透明白色，不抢眼但可见
- 材料选择器默认就停在"cobblestone"上（不需要玩家切换）
- 第一个需求的材料就是默认选中的材料

**成功判定**：玩家成功投放第1个方块

**成功后**：
- NPC："嗯，继续继续，别停！"
- 操作提示缩小为更小的文字（但不消失）

### Step 2: 理解进度（0:30 - 1:00）

**触发**：玩家投放了3个以上方块

**NPC台词**：
```
"才3个？还差7个呢！"
```

**教学目标**：让玩家注意到进度条和计数器

**细节**：
- 气泡中的进度条在这个阶段会有一个短暂的"呼吸"动画（边框闪烁），引导视线
- 只闪烁一次，之后正常

### Step 3: 等待建造（~1:00）

**触发**：10个圆石全部投放完毕

**NPC台词**：
```
"总算好了！看我的！"
```

**教学目标**：让玩家知道"材料够了NPC会自动建造"

**细节**：
- NPC开始建造，头顶显示锤子图标
- 玩家这段时间无事可做 — 这是设计好的"教学呼吸空间"
- 如果玩家这时候乱丢方块，不会有负面后果

### Step 4: 学习生成NPC（~1:20）

**触发**：第一座建筑完成

**NPC台词**：
```
"太无聊了！给我找个伙伴来！"
```

**屏幕提示**（左下角出现）：
```
按 [N] 生成NPC
```

**细节**：
- [N] 键在屏幕上高亮闪烁，直到玩家按下
- 如果玩家10秒内没按N，NPC追加："[N]键！按[N]键啊！你不识字吗？"

### Step 5: 学习切换材料（~1:40）

**触发**：新NPC出现后，两个NPC同时喊食物需求

**NPC台词**：
```
NPC1: "饿死了！给我吃的！"
NPC2: "我也要！苹果！"
```

**屏幕提示**：
```
按 [9] 或 [←→] 切换到苹果
```

**教学目标**：学会切换材料类型

**细节**：
- 如果玩家15秒内没切换到苹果，NPC会指导性地骂："按[9]切到苹果！！听到没！！"
- 投放第一个苹果后提示消失

### Step 6: 教学完成

**触发**：食物需求满足

**标志**：
- 所有操作提示永久消失
- 玩家已学会：投放、进度、等待建造、生成NPC、切换材料
- 从此进入正常游戏流程

## 保护期

### 前60秒的特殊规则

```lua
TUTORIAL_PROTECTION = {
    duration = 60,           -- 保护期60秒
    hungerDecayMult = 0.3,   -- 饥饿衰减只有30%
    tempDecayMult = 0.3,     -- 温度衰减只有30%
    -- 保护期结束后平滑过渡到正常值（10秒内渐变）
}
```

这确保玩家在学习操作时不会因为"刚学会怎么扔东西NPC就饿死了"的尴尬体验。

### 保护期的平滑过渡

```lua
function getTutorialMultiplier(gameTime)
    if gameTime < TUTORIAL_PROTECTION.duration then
        return TUTORIAL_PROTECTION.hungerDecayMult
    elseif gameTime < TUTORIAL_PROTECTION.duration + 10 then
        -- 10秒内从0.3渐变到1.0
        local t = (gameTime - TUTORIAL_PROTECTION.duration) / 10
        return TUTORIAL_PROTECTION.hungerDecayMult + (1 - TUTORIAL_PROTECTION.hungerDecayMult) * t
    else
        return 1.0
    end
end
```

## 非首次玩家

成就系统记录玩家是否曾经完成过教学。第二次游戏开始时：
- 操作提示仍然显示（万一玩家忘了），但更小更淡
- 保护期缩短为30秒
- NPC的教学性台词（如"按[N]键啊"）仍然会出现（它本身就是台词的一部分，不仅仅是教学）

## 边界情况

1. **玩家在教学期间按了不相关的键** — 忽略，不提示
2. **玩家在教学期间切换到蚁眼视角** — 允许，但第一个需求的NPC气泡在蚁眼下仍然可见
3. **玩家在教学期间尝试投放错误材料** — NPC喊 WRONG_MATERIAL 台词，间接教学
4. **玩家在教学期间什么都不做** — NPC每10秒重复催促，逐渐升级语气（从不耐烦到暴怒）
